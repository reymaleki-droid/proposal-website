What We Need From Your Team

A successful delivery requires specific inputs, approvals, and coordination from your side. This page outlines exactly what is needed, when, and from whom.

Required Deliverables

These materials must be provided by your team before or during the project.

Deliverable Description When Needed Required For
Product catalog data Complete list of products with names, categories, specifications, and pricing. Provided as spreadsheet or structured format. Discovery phase All packages
Product images High-resolution product photos. Minimum 800x600px, consistent background preferred. At least 1 image per product. Content phase All packages
Brand assets Logo (SVG or high-res PNG), brand colors, any existing brand guidelines. Discovery phase All packages
Company information About text, contact details, addresses, social media links, legal/registration information. Content phase All packages
Product categories Agreed taxonomy for IT equipment, network infrastructure, and CCTV systems. Include sub-categories. Discovery phase All packages
Hosting credentials Access to hosting account (cPanel for A/B, VPS access for C/E), domain registrar. Build phase All packages
CRM credentials Admin access to CRM platform. API keys, webhook URLs, user permissions. Build phase C, E only
Telegram bot token Bot created via @BotFather. Token and admin Telegram accounts provided. Build phase D, E only
Staff contact list Names and roles for inquiry routing (sales, technical, general). Required for notification setup. Build phase C, D, E

Approval Checkpoints

At each checkpoint, we present deliverables for review. Your team has a 3 business day response window. Delayed responses shift all subsequent milestones by the same duration.

Requirements Sign-off

Review the documented requirements and confirm they accurately capture your needs. This is the foundation — changes after this point may affect timeline and scope.

Who approves: Owner / GM

Design Approval

Review wireframes and visual mockups. Two revision rounds are included for B, C, and E. Approval means the visual direction is locked. Not applicable for Package A.

Who approves: Owner / GM + Sales Manager

Alpha Review

Core features demonstrated on staging environment. Test product listing, filtering, and inquiry flow. Report issues within the response window.

Who approves: IT Manager + Sales Manager

Content Approval

All product data loaded and displayed. Review for accuracy, categorization correctness, image quality, and specification completeness.

Who approves: Sales Manager + IT Manager

Integration Sign-off

CRM sync verified — test lead creation, product sync, notification routing. (C and E only.) Bot commands tested end-to-end (D and E only).

Who approves: Owner / GM + IT Manager

Go-Live Approval

Final sign-off on staging environment. DNS switchover proceeds only after written confirmation. Post-launch monitoring for 1 week with priority support.

Who approves: Owner / GM

Communication Cadence

Weekly Updates

Written progress report delivered every Monday morning. Includes: completed work, upcoming tasks, blockers, and any decisions needed.

Feedback Channel

Dedicated communication channel (email thread or Telegram group) for questions, feedback, and file sharing. Responses expected within 1 business day for non-blocking items.

Checkpoint Meetings

30-minute video or in-person meetings at each approval checkpoint. Scheduled 2 days before the response deadline to allow for clarification.

Escalation Path

If a blocker is unresolved for 5+ business days, the project manager escalates to the Owner/GM directly. Timeline impact is documented and communicated.

Change Request Policy

Changes after requirements sign-off are handled through a structured process.

Change Type Examples Impact Process
Minor content Fix typo, update phone number, swap product image No timeline impact Submit via feedback channel. Handled within 1 business day.
Moderate scope Add a product category, change inquiry form fields, add a static page +1–3 days Documented change request. Timeline impact assessed within 24 hours. Requires written approval.
Major scope Add new integration, change CRM vendor, add multi-language support +1–4 weeks Formal scope amendment. Revised timeline and cost estimate. Requires Owner/GM sign-off.
Architectural Switch from WordPress to custom (A/B to C/E), add e-commerce capability Project restart Treated as a new project. Current work may not carry forward. See migration table on compare page.
